# 담벼락 서비스 PRD (Product Requirements Document)

## 1. 개요 (Product Overview)
본 프로젝트는 누구나 자신만의 온라인 담벼락을 만들고, 방문자들이 포스트잇 형태로 익명(또는 닉네임)의 메모를 남길 수 있는 소셜 방명록 서비스입니다.
Vercel을 통해 프론트엔드를 호스팅하고, Supabase를 백엔드(데이터베이스)로 사용하여 빠르고 안정적인 서비스를 제공합니다.

## 2. 기술 스택 (Tech Stack)
- **Frontend**: Next.js (React), Tailwind CSS (UI 스타일링), Vercel (호스팅)
- **Backend / DB**: Supabase (PostgreSQL, 인증 불필요/익명 접근, 데이터 저장)

## 3. 핵심 기능 요구사항 (Core Features)

### 3.1 담벼락(Wall) 생성 및 공유
- **담벼락 만들기**: 사용자는 홈 화면에서 '담벼락 이름'을 입력하여 즉시 새로운 담벼락을 생성할 수 있습니다.
- **고유 식별자**: 생성된 담벼락은 UUID 기반의 고유한 URL 경로(예: `/wall/[id]`)를 부여받습니다.
- **공유하기**: 담벼락 소유자는 부여받은 URL을 복사하여 다른 사람들에게 공유할 수 있습니다.

### 3.2 포스트잇(낙서) 작성
- **작성 UI**: 담벼락 페이지 내에 눈에 띄는 '작성하기' 버튼이 존재하며, 클릭 시 모달 또는 폼이 나타납니다.
- **입력 항목**:
  1. 닉네임 (선택사항, 미입력 시 '익명')
  2. 내용 (낙서 내용)
  3. 비밀번호 (필수, 추후 본인이 작성한 메모를 삭제하기 위한 용도)
  4. 포스트잇 색상 (노랑, 분홍, 하늘, 연두 등 선택)
- **데이터 저장**: 폼 제출 시 Supabase 데이터베이스에 포스트잇 데이터가 안전하게 저장됩니다.

### 3.3 포스트잇 조회 (담벼락 렌더링)
- **시각화**: 담벼락 페이지 접속 시, 데이터베이스에 저장된 포스트잇들이 화면에 렌더링됩니다.
- **레이아웃**: 실제 포스트잇처럼 보이도록 Masonry(조적식) 레이아웃이나 격자(Grid)로 다채롭게 배치합니다.
- **디자인 감성**: 섀도우(Drop shadow) 효과, 아기자기한 폰트 등을 활용해 실제 다이어리를 꾸미는 듯한 느낌을 줍니다.

### 3.4 포스트잇 삭제
- **삭제 권한**: 포스트잇 우측 상단의 '삭제' 아이콘이나 버튼을 클릭하면 **비밀번호 입력창**이 나타납니다.
- **조치**: 등록 시 설정한 비밀번호와 일치하는 경우에만 해당 포스트잇이 데이터베이스 및 브라우저에서 영구 삭제됩니다.

## 4. 데이터베이스 스키마 (Supabase)

**`walls` (담벼락)**
- `id`: uuid (Primary Key)
- `title`: text (담벼락 이름)
- `created_at`: timestamp (생성 시간)

**`postits` (포스트잇)**
- `id`: uuid (Primary Key)
- `wall_id`: uuid (Foreign Key -> walls.id)
- `nickname`: text (작성자 닉네임)
- `content`: text (등록된 메모)
- `password`: text (삭제를 위한 비밀번호)
- `color`: text (배경색 지정 정보)
- `created_at`: timestamp (작성 시간)

## 5. UI/UX 요구사항
- 모바일 프렌들리: 반응형 웹 설계를 바탕으로 모바일 화면에서도 포스트잇들이 예쁘게 배열되어야 합니다.
- 직관적인 사용: 회원가입-로그인 절차를 배제하여 사용자 이탈을 최소화합니다.
